FORMS HANDLING :
----------------
refers to the process of managing user input from HTML forms with react applications, ensuring data integrity and a responsive user experience.

we can handle form in react in two ways:
1.controlled form : each and every form input managed by react state. we can create CF using usestate hook. 
	the form data is stored in the components state using USESTATE HOOK
2.uncontrolled form : in react a way of managing form data where the form elements maintain their own state.
 =>instead of using react state control the input values, uncontrolled components rely on the DOM to handle the form data.
 =>we can create UNF using USEREF HOOKS.

USE REF
=======
Ur is a hook provided by react only which is used to create a mutable refrence that persists across renders without causing re-renders.
it access and manipulate DOM elements such has focusing an input field.
it stores previous state values without trigering re renders.
it keeps drag of intervals, timers or event listeners.

syntax:
--------------------------------------------
import { useRef } from "react";             |
                                            |
const refContainer = useRef(initialValue);  |
--------------------------------------------



-----------------------------------------------------------------------------------------------------------------------
 DIFF BETWEEN USE REF AND USE STAE                  |                                                                   |
USE REF : * IT doesnot rerender the componet        |     use stste: * it renders the componet based on state change.  |
          * it is used for dom manupulation.        |               * it doesnot change the value from directly dom.  |

CONTEXT API
************
=>the context API is a feature in react that enables sharing data between components without the need of explicitly pass props downthrough every levelof components.
this is a problem referd as prop-drilling to avoid this onle we have context api.
=> it provides a way to manage and access global or shared state across different part of your applications.
=> it i a feature , which can used to pass the data to child directly.

Key Features
------------
1.avoid prop drilling
2.global data sharing
3.dynamic updates

UseCase:
--------
1.theming
2.user authentication
3.language switching

HOW TO SHARE THE DATA GLOBALLY
******************************
=> to share the data globaly first we need to create context.
=> create context is a menthod provided by react. in this we get our provider and consumer.
=> provider is responsible for passing the data.
=> to receive this data we need to make the component  as a children for provider.
=> we can get the data with the help of useContext.
=> here we need to pass ou created context

Here we have 2 things:
......................
1.provider
2.consumer

=>

benefits:
--------
1.code maintenence
2.reusability
3.readability


USEEFFECT:
**********
=> it is a hook in react that enable functional component to perform side effects.
=> side effects are operations that interact with outside react such as fetching datas directly updating the dom setting 
subscriptions or event listeners etc.

which allows us to perform side effects: means fetching data.

HOW YOUR USE EFFECT WORKS:
==========================
* it accept one function like callback function
* this function contains the code for your side effects.
* react will execute this function after every render by default or based on the specific dependencies.

DEPENDENCIES
============



HOW MANY WAYS:
************

1.with dependency([a]):-
------------------------
executes the effects only after provided dependency changes

2.empty dependency([]):-
------------------------
executes the effect only one time

3.without dependency:-
-----------------------
executes the effect after every render.


CLEAN UP FUNCTION:(Optional)
----------------------------

* the fun passed to use effect can optional return another function.
* this return function is a cleanup function which react will execute before the component unmounts.
or before the effect reruns due to changed dependencies.
* it is crusial for preventing memory links.

MODULE CSS
***********
which is used to style the files/component seperately.
=>in global css we have a single css file for all the component but in module css we have a seperate css files for each and every component.
=>css module are a mordern approach to styling in web dvelopment, designed to solve issues related to global class name conflicts and 
style leakage.

FEATURE :
=========
=>Automatic Scope Management: With CSS Modules, styles are scoped to the component by default. This reduces the risk of styles accidentally affecting other parts of the app, which is especially helpful when multiple developers are working on different features.
=>Unique Class Names: Tools like Webpack and PostCSS automatically generate unique, hashed class names during the build. This takes the burden off developers from having to come up with naming conventions (like BEM) and reduces the chances of mistakes.
=>Easier Debugging: Since the class names are scoped and unique, itâ€™s much easier to identify where a styling issue is coming from. Tools like React DevTools make it even simpler by linking CSS Modules to their corresponding components.
=>Smooth Integration with JavaScript Frameworks: CSS Modules work seamlessly with modern JavaScript frameworks like React, Vue, and Svelte. You can import your styles directly into the components, making it easier to manage both the structure and appearance in one place.
=>Better Performance: Since CSS Modules only apply styles to the relevant components, they help minimize unused CSS in the final build. This, along with techniques like tree-shaking and CSS purging, can lead to performance improvements.
=>Fewer Global Side Effects: By scoping styles to individual components, CSS Modules prevent the usual problems with global styles interfering with each other. This makes it much easier to build reusable UI components without worrying about conflicts.

import React from 'react'
import { createBrowserRouter, RouterProvider } from 'react-router-dom'
import Layout from './Layout'
import Contact from './Components/Contact'
import Login from './Components/Login'
import Home from './Components/Home'
import About from './Components/About'

const App = () => {
    const router = createBrowserRouter([
        {
            path:'/',
            element: <Layout/>,
            children:[
                {
                    path:'/',
                element:<Contact/>
                }
                
            ]     
        },
        {
            path:'/home',
            element:<Home/>
        },
        {
            path:'/about',
            element:<About/>
        },
        {
            path:'/contact',
            element: <Contact/>
        },{
            path:'/login',
            element:<Login/>
        },
    ])
  return <RouterProvider router={router}/>

}

export default App

AXIOS
=======
=>axios is used to do http operations in react js.
=>it simplifies the process of sending asynchronous http requests to rest endpoints, handling responses and performing various n/w related tasks.

Key Features of Axios
=>Promise-Based: Axios uses Promises, making it easier to handle asynchronous requests with modern JavaScript features like async/await.
=>Interceptors: Allows you to intercept and modify requests or responses before they are handled by .then() or .catch().
=>Automatic JSON Data Transformation: Automatically transforms JSON data to JavaScript objects, making data manipulation simpler.
=>Request and Response Timeout: Enables setting timeouts to avoid hanging requests.
=>Cancel Requests: Allows cancellation of requests to manage resources effectively.
=>Error Handling: Provides a structured way to handle HTTP errors using the catch() method.
=>Supports All HTTP Methods: Including GET, POST, PUT, DELETE, PATCH, and more.
=>Cross-Site Request Forgery (CSRF) Protection: Easily supports security features like CSRF by managing tokens.
